# Site — Full Functionality Overview

All remaining features: auth, navigation, catalog, product page, QuickView, homepage, search.

## 1. Authentication — Login Flow

Data source: `signInAction` (Server Action) → OneEntry Platform `AuthProvider.auth('email', …)`. Session lives in httpOnly cookies (`oe_access`, `oe_refresh`, `oe_user`) — nothing sensitive in Redux or localStorage.

<MermaidDiagram chart={`flowchart TD
    A(["Click Sign In\\n(Header or Guest Modal)"]) --> B["LoginModal opens\\nfocus trap active\\nbody scroll locked"]
    B --> C{"Fill email/phone\\n+ password"}
    C --> D["Zod loginSchema\\nclient-side check"]
    D -->|invalid| E["Show field error\\ndo not submit"]
    D -->|valid| F["AuthContext.login\\ncalls signInAction"]
    F --> F2["OneEntry\\nAuthProvider.auth\\nmarker email"]
    F2 -->|"4xx"| G["error: Wrong credentials"]
    F2 -->|"200"| G2["Server sets oe_access,\\noe_refresh, oe_user cookies"]
    G2 --> H["AuthContext seeds user\\nfrom getCurrentUserAction\\nsetIsLoggedIn true"]
    H --> I["closeLoginModal\\nno redirect"]
    H --> J["CartContext + WishlistContext\\ndebounced 400 ms sync effects\\nunlock"]
    H --> K["dispatch setAuth\\nuserIdentifier only\\ntokens stay empty"]
    B --> L{"Continue with Google"}
    L -->|"NEXT_PUBLIC_GOOGLE_CLIENT_ID set"| M["startGoogleOAuth returnTo\\nbrowser redirects to\\nGoogle authorization endpoint"]
    L -->|"unset"| N["Button hidden"]
    M --> M2["User consents on Google"]
    M2 --> M3["Redirect to\\napp/auth/callback/google/route.ts\\nwith authorization code"]
    M3 --> O["Server: exchange code for tokens\\nthen signInWithGoogleAction accessToken"]
    O --> F2
    B --> P{"Apple / Facebook"}
    P --> Q["Stub — password=social\\nsets empty user session"]
    style Q fill:#fffbeb,stroke:#fde68a,color:#92400e`} />

Demo credentials: run `./scripts/setup-demo-passwords.sh` once; every `seed-demo-user-*` identifier accepts `demo123`.

## 2. Authentication — Register Flow

Sign-up form is CMS-driven — fields, placeholders, and validators come from the `users_sign_in_sign_up` attribute set loaded by `loadSignUpFormSchema` and exposed via `SignUpFormSchemaProvider`.

<MermaidDiagram chart={`flowchart TD
    A(["Click Create Account"]) --> B["RegisterModal reads\\nSignUpFormSchemaContext"]
    B --> C["Fill: email, password,\\nfirst_name, phone, gender,\\nsubscriptions, terms"]
    C --> D["Zod registerSchema\\nclient check"]
    D -->|invalid| E["Per-field errors shown"]
    D -->|valid| F["AuthContext.signUp\\ncalls signUpAction"]
    F --> G["OneEntry\\nAuthProvider.signUp\\nform=signin"]
    G -->|"4xx"| H["Error surfaced verbatim"]
    G -->|"200"| I["Cookies set +\\nuser seeded +\\nsetIsLoggedIn true"]
    I --> J["closeRegisterModal"]`} />

No email verification step is wired on this tenant.

## 12. Recently Viewed

Data source: `state.recentlyViewed.items` (Redux). Max **100** items (`LIMITS.RECENTLY_VIEWED_MAX`), 30-day TTL, persisted in localStorage. Signed-in users mirror the trail to OneEntry via three Server Actions.

<MermaidDiagram chart={`flowchart LR
    A(["Open any product page"]) --> B["dispatch recentlyViewedActions.addProduct\\nviewedAt = Date.now()"]
    B --> C["Dedup by product name\\nevict items older than 30 days"]
    C --> D["Circular buffer, cap 100"]
    D --> E["saveToStorage\\nlocalStorage oe_store.recentlyViewed"]
    B -->|"if isLoggedIn"| F["pushRecentlyViewedAction\\n(fire-and-forget)"]
    F --> G["Server trail updated"]
    H(["Bootstrap on login"]) --> I["mergeRecentlyViewedAction\\nguest + server → deduped list"]
    I --> J["dispatch recentlyViewedActions.hydrate"]
    E --> K["RecentlyViewedSection\\nlazy-loaded via IntersectionObserver\\nshown below PDP, excluding current"]`} />
