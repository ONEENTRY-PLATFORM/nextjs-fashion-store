# Redux — Server Actions

The storefront's Redux slices are the client-side optimistic layer. Everything server-authoritative — auth, cart / wishlist / recently-viewed sync, profile mutations, order creation, form submits, activity tracking, catalog / block / page / label / menu reads — happens through **Next.js Server Actions and RSC loaders** under `src/lib/oneentry/**`.

This page inventories what the state layer touches on the server side.

Grouped by domain. See `docs/ONEENTRY_INTEGRATION.md` for full request / response shapes and marker registry.

---

## Storybook impact

Storybook runs without OE env vars — `getApi()` throws when the SDK is touched. In practice:

- `AuthContext` bootstrap fails; `authReady` still flips to `true` but `user` stays `null`, `isLoggedIn` is `false`.
- `syncCart` / `syncWishlist` never fire because the debounce effect gates on `isLoggedIn`.
- Loaders and labels come from the mocked data / prop values passed by each story.

If a story needs to demonstrate a Server Action, stub the module — do not point Storybook at a live Platform.
