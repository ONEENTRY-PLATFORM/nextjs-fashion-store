## catalogSlice

Per-catalog UI state (filters, sort, pagination, view mode). Key = catalog identifier string.

**State shape per catalog key:**

```ts
interface CatalogUIState {
  selectedFilters: Record<string, string[]>; // filterKey â†’ selected values
  sortBy: string;        // default: 'featured'
  currentPage: number;   // default: 1
  viewCols: 3 | 4;       // grid columns, default: 4
  listMode: boolean;     // list vs grid, default: false
  activeChip: string;    // quick filter chip, default: ''
}
// Full state: Record<catalogKey, CatalogUIState>
// Access: useAppSelector(s => s.catalog['women-clothing'])
```

**Imports:** Actions are exported as **named exports**, not a grouped object:

## recentlyViewedSlice

Persisted to `localStorage['oe_store'].recentlyViewed`. Circular buffer, max 100 items, 30-day TTL.

**Stored type:** `RecentlyViewedItem = Product & { viewedAt: number }`

> Used in `ProductDetailPage` â€” dispatches `addProduct` on mount, renders "Recently Viewed" block via `<RecentlyViewedSection>`.

## userSlice

Default state seeded from `USER_DATASET` in `data/userData.ts`. Real profile data comes from `AuthContext.user` (populated by `getCurrentUserAction` â€” see `05-Contexts`). **This slice is NOT persisted to localStorage** â€” the session lives in httpOnly cookies (`oe_access`, `oe_refresh`, `oe_user`), managed by Server Actions.

The `authToken` / `refreshToken` fields are retained on `state.user.data` for backward-compatibility but are kept as empty strings on the live path. They are legacy from an older client-side auth model and will be pruned in a future cleanup.
