# Fashion E-Commerce UI Style Guide  
*Prompt for Figma Maker*

Create a modern fashion e-commerce interface following 2025 UI/UX trends: minimalism, strong typography hierarchy, generous whitespace, high contrast CTAs, soft micro-interactions, and clean grid-based layouts.

Use a refined, editorial aesthetic inspired by premium fashion brands.

---

# 1. Typography System

## Font Family
**Inter** (Primary UI font)  
- Clean, modern grotesk  
- High legibility  
- Used across all components  

---

- Weight: 600 (Semibold)  
- Size: 48px  
- Line height: 56px  
- Letter spacing: -0.5px  

### H2
- Font: Inter  
- Weight: 600  
- Size: 32px  
- Line height: 40px  

### H3
- Font: Inter  
- Weight: 600  
- Size: 24px  
- Line height: 32px  

### Body Large
- Font: Inter  
- Weight: 400  
- Size: 18px  
- Line height: 28px  

### Body Regular (Base Text)
- Font: Inter  
- Weight: 400  
- Size: 16px  
- Line height: 24px  

### Small / Meta
- Font: Inter  
- Weight: 400  
- Size: 14px  
- Line height: 20px  

---

## Mobile Typography Scale

### H1
- 32px / 40px  
- Weight: 600  

### H2

- Weight: 600

- Weight: 600

# 3. Button System

Buttons must follow a modern, flat, high-contrast style.  
Rounded corners: 8px–12px radius.  
No heavy shadows.  
Subtle hover transitions (150–200ms ease-in-out).

---

## Primary Button

### Default
- Background: Primary Accent  
- Text: White  
- Border: None  

### Hover
- Slightly darker accent (−8% brightness)  
- Smooth background transition  

### Active / Pressed
- Darker accent (−15%)  
- Slight scale: 0.98  

### Disabled
- Background: #E5E5E5  
- Text: #9E9E9E  
- No hover effect  

---

## Secondary Button (Outline)

### Default
- Background: Transparent  
- Border: 1px solid Black  
- Text: Black  

### Hover
- Background: Black  
- Text: White  

### Disabled
- Border: #E5E5E5  
- Text: #BDBDBD  

---

## Ghost Button

### Default
- Background: Transparent  
- Text: Black  

### Hover
- Background: rgba(0,0,0,0.05)

# 4. Product Card Buttons

Each product card includes two button types:

---

## 4.1 Static Buttons

Used for informational or secondary actions.

Examples:
- Wishlist icon (heart)
- Compare icon
- Color swatches
- Size chips (default state)

Style:
- Minimal
- Outline or icon-only
- Subtle hover state
- No heavy fill

---

## 4.2 Interactive Buttons

Primary action buttons requiring strong emphasis.

Examples:
- Add to Cart
- Quick View
- Select Size
- Buy Now

Style:
- Use Primary Button system
- Strong contrast
- Clear hover and active states
- Full-width on mobile

---

# 5. Modern Trend Guidelines

- Generous whitespace (8px base spacing system)
- 12-column grid desktop
- 4–8px spacing increments
- Soft motion (fade + slide)
- No gradients (unless subtle background usage)
- High accessibility contrast
- Clean product-first design

The overall interface must feel:
- Premium
- Clean
- Confident
- Fashion-forward
- Editorial yet functional
